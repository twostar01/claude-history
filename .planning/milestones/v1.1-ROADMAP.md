# Milestone v1.1: Search & Ingest Improvements

**Status:** ✅ SHIPPED 2026-05-18
**Phases:** 5–6
**Total Plans:** 2

## Overview

Fill the gaps left by v1.0 — richer search filtering, correct incremental ingest, and file export capability. All four targeted requirements shipped and verified live; ATTACH-01 (PDF indexing) intentionally deferred.

**Depends on**: Phase 4
**Requirements**: SRCH-01, SRCH-02, EXP-01
**Plans**: 1 plan

Plans:

- [x] 05-01: Extend search.py (role/date filters) and server.py (tool wrappers + export file write)

**Details:**

Success criteria met:
1. User can call `search_conversations` with `date_from` and/or `date_to` (ISO date strings) and receive only results within that window
2. User can call `search_conversations` with `role_filter="human"` or `role_filter="assistant"` and receive only results from messages sent by that role
3. User can call `export_conversation` with a `file_path` argument and find a valid markdown file written to that path

## Milestone Summary

**Key Decisions:**

- `[:10]` prefix comparison for ISO timestamp date filtering — avoids T-separator boundary failure where `'2026-03-05T...' > '2026-03-05'` would silently exclude same-day conversations
- Role filter applied as SQL `AND m.role = ?` parameterized predicate in the FTS JOIN — no Python post-processing, no injection surface
- Date filter applied post-BM25-dedup, pre-limit-slice via single bulk `SELECT id, created_at FROM conversations WHERE id IN (...)`
- `file_path=None` preserves existing export_conversation behavior — fully backward compatible
- `export_conversation` returns `"Written to: {Path.resolve()}"` for machine-readable absolute path on Windows
- Removed early-exit UUID check (`SELECT 1 FROM conversations WHERE id = ?` + `continue`) that was silently skipping all messages for pre-existing conversations
- `total_new_msgs` scoped to existing-conversation updates only so "M updated (K msgs)" log line is internally consistent

**Issues Resolved:**

- INGEST-01 root cause: the early-exit skip in `ingest_zip()` meant re-runs silently dropped all new messages for conversations already in the DB
- BM25 dedup + date filter ordering: date filter must run after dedup but before limit-slice to avoid under-returning results

**Issues Deferred:**

- ATTACH-01: PDF attachment indexing deferred from v1.1; requires pypdf/pdfplumber; narrow value for personal use

**Technical Debt Incurred:**

- `project_filter` parameter kept in `search_conversations` tool signature for schema compatibility but not applied (Claude.ai export has no project-conversation association)
