## v1 Requirements — All Complete

### Ingestion

- [x] **INGEST-01**: User can run a schema discovery script against a Claude.ai export ZIP that prints the JSON field structure without modifying the database — **Phase 1** ✅
- [x] **INGEST-02**: User can run the ingest script against a Claude.ai export ZIP to load all conversations and messages into SQLite — **Phase 2** ✅ (106 conversations, 4087 messages)
- [x] **INGEST-03**: Re-running ingest on the same or updated export does not create duplicate conversations (INSERT OR IGNORE + UUID pre-check) — **Phase 2** ✅ (verified: 0 new, 106 skipped)
- [x] **INGEST-04**: Ingest script skips conversations already in the database when running incrementally — **Phase 2** ✅
- [x] **INGEST-05**: Ingest script indexes content of text and code attachments (.txt, .py, .js, .md, etc.) alongside conversation messages; skips binary files gracefully — **Phase 2** ✅ (41 conversations with attachment content)

### Database

- [x] **DB-01**: SQLite database uses FTS5 virtual table with unicode61 tokenizer (`tokenchars '-_'`) enabling snake_case-aware search — **Phase 2** ✅
- [x] **DB-02**: FTS5 table uses content table pattern (no duplicate text storage) with AFTER INSERT / AFTER DELETE / AFTER UPDATE triggers to stay in sync — **Phase 2** ✅
- [x] **DB-03**: Database uses WAL journal mode to allow concurrent reads during ingest — **Phase 2** ✅
- [x] **DB-04**: Schema stores conversations (id, title, project, created_at, message_count) and messages (id, conversation_id, role, content, position, created_at) — **Phase 2** ✅

### MCP Tools

- [x] **TOOL-01**: `search_conversations(query, project_filter?)` returns matching conversations with BM25-ranked snippets, title, project, date, and match count — **Phase 3** ✅
- [x] **TOOL-02**: `search_conversations` accepts `include_full_content=true` flag to return complete message text instead of snippets — **Phase 3** ✅
- [x] **TOOL-03**: `get_conversation(id)` returns full conversation content formatted as labeled turns (Human/Assistant) — **Phase 3** ✅
- [x] **TOOL-04**: `list_projects()` returns all project names with conversation counts and date ranges — **Phase 3** ✅ (returns [] — export format limitation, documented in tool docstring)
- [x] **TOOL-05**: `get_stats()` returns total conversation count, total message count, date range of indexed data, and database file size — **Phase 3** ✅
- [x] **TOOL-06**: `export_conversation(id, format?)` returns the conversation formatted as a markdown string suitable for pasting or summarizing — **Phase 3** ✅

### Server & Integration

- [x] **SETUP-01**: Project is structured as a uv package with `pyproject.toml`, entry points for `server` and `ingest` commands — **Phase 1** ✅
- [x] **SETUP-02**: Server uses FastMCP with stdio transport; all logging goes to stderr; stdout is never written to directly — **Phase 3** ✅
- [x] **SETUP-03**: MCP server is registered with Claude Code via `claude mcp add -s user` so it is available in every session — **Phase 4** ✅
- [x] **SETUP-04**: `.gitignore` excludes the SQLite database file and any Claude.ai export ZIPs — **Phase 1** ✅
- [x] **SETUP-05**: README documents installation steps, ingest workflow, and Claude Code registration command — **Phase 4** ✅
