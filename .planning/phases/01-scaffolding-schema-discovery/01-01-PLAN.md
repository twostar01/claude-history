Step 1 — Install uv (it is NOT currently in PATH on this machine):

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

After install, reload the shell PATH so `uv` is available. Verify: `uv --version` prints a version string.

Step 2 — Initialize the uv package (already inside the repo root at C:\Users\nclem\Claude Code\claude-history):

```powershell
uv init --package . --no-readme
```

The `--package` flag is REQUIRED — without it there is no `[build-system]` section and `[project.scripts]` entry points will not work. The `--no-readme` flag prevents overwriting any existing README.md.

If `uv init` refuses because pyproject.toml already exists, create it manually using the exact template below.

Step 3 — Install mcp dependency:

```powershell
uv add "mcp[cli]>=1.2.0"
```

This creates `.venv/`, `uv.lock`, and adds `mcp[cli]` to `[project.dependencies]`.

Step 4 — Verify the generated pyproject.toml has [build-system]. If uv init generated it correctly, it will already contain:
```toml
[build-system]
requires = ["uv_build>=0.11.8,<0.12"]
build-backend = "uv_build"
```

If the build-system section is absent (uv init without --package falls through), add it manually.

Step 5 — Add [project.scripts] entry points to pyproject.toml. The final pyproject.toml MUST contain this section exactly:

```toml
[project]
name = "claude-history"
version = "0.1.0"
description = "MCP server for searching Claude.ai conversation history"
requires-python = ">=3.11"
dependencies = [
    "mcp[cli]>=1.2.0",
]

[project.scripts]
server = "claude_history.server:main"
ingest = "claude_history.ingest:main"
schema-discovery = "claude_history.schema_discovery:main"

[build-system]
requires = ["uv_build>=0.11.8,<0.12"]
build-backend = "uv_build"
```

Note: `ingest` and `schema-discovery` point to modules that do not exist yet (Phase 2 and Plan 01-03). That is fine — uv does not validate entry point targets at install time.

Step 2 — Create `src/claude_history/config.py` with this exact content:

```python
"""Configuration constants for the claude-history MCP server."""

from pathlib import Path

# Resolve DB_PATH relative to this file so it works regardless of working directory.
# src/claude_history/config.py is 3 levels below the project root:
#   config.py -> claude_history/ -> src/ -> project root
DB_PATH: Path = Path(__file__).parent.parent.parent / "history.db"
```

The `.parent.parent.parent` chain is critical: `Path(__file__)` is `src/claude_history/config.py`, so `.parent` is `src/claude_history/`, `.parent.parent` is `src/`, and `.parent.parent.parent` is the project root where `history.db` will live.

Step 3 — Verify the path resolves correctly:

```powershell
uv run python -c "from claude_history.config import DB_PATH; print(DB_PATH)"
```

The output must end with `history.db` and be an absolute path starting with the project root (e.g., `C:\Users\nclem\Claude Code\claude-history\history.db`).
